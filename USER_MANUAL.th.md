## 1. การติดตั้งและเริ่มต้นโปรเจกต์ใหม่

ก่อนที่จะให้ AI เริ่มเขียนโค้ดในโปรเจกต์ของคุณ คุณต้องติดตั้ง Protocol เพื่อกำหนด "กรอบ" ให้ AI ซะก่อน

**ขั้นตอน:**
1. เข้าไปที่โฟลเดอร์โปรเจกต์ของคุณ (เช่น `cd ~/my-new-app`)
2. รันคำสั่ง `init` โดยชี้ไปยังโฟลเดอร์ที่คุณเก็บ AI Protocol ไว้:
   ```bash
   /path/to/ai-coding-protocol/ai-protocol.sh init
   ```
3. เมื่อรันเสร็จ สิ่งที่คุณจะได้รับคือ:
   - โฟลเดอร์ `.ai/` ซึ่งบรรจุไฟล์ความจำ (`STATE.md`, `REFLECTIONS.md`, `DECISIONS.md`)
   - ไฟล์กฎสำหรับ IDE ต่างๆ เช่น `.cursorrules`, `.clinerules`, `.windsurfrules`
   - เครื่องมือสคริปต์ `ai-protocol.sh` ไว้ใช้ในโปรเจกต์ของคุณ

**ข้อแนะนำ:** หลังจากรัน `init` แล้ว แนะนำให้รันคำสั่งติดตั้ง Git Hook เพื่อความปลอดภัย:
```bash
./ai-protocol.sh install-hook
```
คำสั่งนี้จะช่วยตรวจสอบความเรียบร้อยทุกครั้งก่อนที่คุณจะสั่ง `git commit`

## 2. การตั้งค่าดีไซน์ UI ของโปรเจกต์

เพื่อไม่ให้ AI สุ่มสี สุ่มฟอนต์ หรือออกแบบมั่วซั่ว คุณต้องกำหนด "Design System" ให้ AI ยึดเป็นหลักการเดียว (Single Source of Truth)

**ขั้นตอน:**
ระบบมีเทมเพลตมาตรฐานให้เลือก 4 แบบ คุณสามารถคัดลอกอันที่ชอบมาทับไฟล์เดิมได้เลย:

1. **Futuristic Space-Dark** (อวกาศล้ำๆ, ใช้โทนดำ, กระจกใส) - *เป็นค่า Default*
   ```bash
   cp .ai/templates/ui/futuristic.md .ai/docs/ui_guidelines.md
   ```
2. **Minimalist & Clean** (สไตล์ shadcn/ui เรียบหรู, ขาว-ดำ)
   ```bash
   cp .ai/templates/ui/minimal.md .ai/docs/ui_guidelines.md
   ```
3. **Modern Vibrant** (สไตล์ HeroUI/NextUI, มีสีสันสดใส)
   ```bash
   cp .ai/templates/ui/vibrant.md .ai/docs/ui_guidelines.md
   ```
4. **Data-Heavy Dashboard** (สไตล์ Tremor, เน้นกราฟและข้อมูล)
   ```bash
   cp .ai/templates/ui/data_heavy.md .ai/docs/ui_guidelines.md
   ```

หลังจากนี้ AI จะเข้าไปอ่านไฟล์ `.ai/docs/ui_guidelines.md` เสมอก่อนเขียน CSS หรือ Tailwind Component

## 3. วงจรการทำงานกับ AI (Workflow ประจำวัน)

การทำงานกับ AI ในโปรโตคอลนี้จะเน้นไปที่การ **"Micro-Stepping"** และไม่เปิดแชทยาวเกินไป เพื่อประหยัด Token และลดอาการ Hallucination (AI หลอนมั่วโค้ด)

**ขั้นตอนที่ถูกต้อง:**
1. **เริ่มเซสชันใหม่:** เปิดแชทใหม่ใน Cursor หรือ IDE ของคุณ แล้วก๊อปปี้ข้อความจากไฟล์ `.ai/prompts/01-session-start.md` ลงไปวาง เพื่อให้ AI รับรู้สถานะปัจจุบัน
2. **สั่งงาน:** สั่งงานตามปกติ แต่ **ห้ามสั่งให้ทำหลายอย่างพร้อมกัน** ให้ AI ทำทีละฟีเจอร์ย่อย (แก้ไฟล์ไม่เกิน 2-3 ไฟล์ต่อรอบ)
3. **ทดสอบก่อนไปต่อ:** ถ้า AI พิมพ์ว่า `[WAITING_FOR_USER]` ให้คุณลองรันโปรแกรมดูว่าเวิร์คไหม ถ้าเวิร์คก็บอกให้ลุยต่อ ถ้าพัง ให้ AI แก้ทันที ห้ามสะสมบั๊ก
4. **ปิดเซสชัน:** เมื่อทำฟีเจอร์เสร็จ ให้ก๊อปปี้ข้อความจาก `.ai/prompts/02-session-end.md` มาแปะ เพื่อสั่งให้ AI อัปเดต `STATE.md` และสรุปงาน

## 3.1 ระบบเมนูลัด IDE Task Menu (ไม่ต้องจำคำสั่ง)

หากคุณไม่ชอบจำหรือพิมพ์คำสั่ง CLI ยาวๆ ในตอนที่รันติดตั้งระบบ (`init`) ระบบจะทำการสร้างไฟล์ `.vscode/tasks.json` ให้โดยอัตโนมัติ 

ทำให้คุณสามารถกดใช้งานคำสั่งทั้งหมดผ่าน **เมนูแบบคลิกเลือก (Dropdown List)** ได้เลย:
1. ไปที่เมนูด้านบนของโปรแกรม (VS Code, Cursor, Antigravity) เลือก **Terminal -> Run Task...**
2. คลิกเลือกคำสั่งที่ต้องการจากรายการที่เด้งขึ้นมาได้เลย (เช่น `🚀 AI Protocol: Dashboard`)

*(ทิปส์: หรือถ้าขี้เกียจกด คุณสามารถ **พิมพ์บอก AI เป็นภาษาไทยในช่องแชท** ให้รันคำสั่งแทนคุณได้เลย เช่น "AI ช่วยเปิดหน้าต่าง dashboard ให้หน่อย")*

## 4. การจัดการความจำของ AI (RAM & Reflexion)

ไฟล์ในโฟลเดอร์ `.ai/` ทำหน้าที่เปรียบเสมือนสมองของ AI คุณต้องคอยดูแลให้มันสะอาดอยู่เสมอ:

- **`STATE.md` (เปรียบเสมือน RAM):** ใช้เก็บว่า "ตอนนี้กำลังทำอะไรอยู่" ห้ามปล่อยให้ยาวเกิน 20 บรรทัด! ถ้าทำงานเสร็จแล้วและจะเริ่มฟีเจอร์ใหม่ ให้รันคำสั่ง:
  ```bash
  ./ai-protocol.sh clean
  ```
  ระบบจะสำรองไฟล์เก่าและสร้าง `STATE.md` อันใหม่ให้คุณ
- **`REFLECTIONS.md` (บทเรียนที่เรียนรู้):** ทุกครั้งที่ AI แก้บั๊กยากๆ สำเร็จ มันจะมาจดไว้ที่นี่เพื่อไม่ให้ทำผิดซ้ำอีก แต่ถ้ามันยาวเกิน 15 ข้อ Token จะบวม ให้คุณรัน:
  ```bash
  ./ai-protocol.sh prune
  ```
  ระบบจะตัดเอาของเก่าไปเก็บไว้ในโกดังถาวรให้
- **`DECISIONS.md` (บันทึกสถาปัตยกรรม):** ห้าม AI ตัดสินใจเลือกเทคโนโลยีเอาเอง (เช่น เปลี่ยนจาก MongoDB เป็น PostgreSQL) AI ต้องถามคุณก่อน และจดบันทึกลงไฟล์นี้

## 5. การส่งมอบงานให้ AI ตัวใหม่ (Handoff)

เมื่อช่องแชทยาวเกินไป AI จะเริ่มเบลอ ทำงานช้า และกิน Token มหาศาล คุณจำเป็นต้อง "ปิดแชทเดิมและเปิดแชทใหม่"

แต่ถ้าเปิดแชทใหม่ AI ก็จะลืมบริบททั้งหมดใช่ไหม? เราแก้ปัญหานี้ด้วยคำสั่ง **Handoff**

**วิธีใช้งาน:**
1. รันคำสั่งใน Terminal:
   ```bash
   ./ai-protocol.sh handoff
   ```
2. ระบบจะทำการสรุป `STATE.md`, `REFLECTIONS.md`, `DECISIONS.md` และไฟล์ที่คุณยังไม่ได้ Commit (Git Status) รวบรวมมาเป็น Prompt ข้อความเดียว
3. ก๊อปปี้ข้อความทั้งหมดที่ระบบสร้างให้ นำไปแปะในหน้าแชท AI ช่องใหม่
4. AI ตัวใหม่จะรับรู้บริบททุกอย่างและทำงานต่อได้อย่างลื่นไหลทันที!

## 6. หน้าปัดควบคุม AI (Dashboard)

หากคุณต้องการดูภาพรวมของโปรเจกต์แบบรวดเร็ว โดยไม่ต้องไปเปิดอ่านไฟล์โฟลเดอร์ `.ai/` ทีละไฟล์ คุณสามารถเรียกใช้งานหน้าปัดควบคุมได้:

```bash
./ai-protocol.sh dashboard
```

**สิ่งที่คุณจะเห็นบน Dashboard:**
- **📍 Pending Tasks:** สรุปงานปัจจุบันและสเต็ปถัดไปที่ AI ค้างไว้ (ดึงจาก `STATE.md`)
- **📚 Recent Reflections:** บทเรียนและข้อผิดพลาดล่าสุดที่ AI บันทึกไว้ เพื่อเตือนความจำ
- **🏗️ Architecture Decisions:** การตัดสินใจสำคัญทางสถาปัตยกรรม (ดึงจาก `DECISIONS.md`)

ทำให้คุณและทีมงานเข้าใจภาพรวมระบบได้ภายใน 1 วินาที!

## 7. ความปลอดภัยเชิงลึก (Atomic Writes & Data Safety)

ระบบ AI Coding Protocol ได้ถูกอัปเกรดกลไก "กันพัง" (Fail-safe) ไว้เบื้องหลัง เพื่อให้ข้อมูลความจำของ AI ปลอดภัยที่สุด:

1. **Transactional Pruning (กันข้อมูลหาย):** เมื่อสั่ง `prune` ระบบจะบันทึกประวัติลงไฟล์ Archive ให้เสร็จสมบูรณ์แบบ 100% ก่อน ถึงจะไปย่อไฟล์ความจำหลัก ป้องกันข้อมูลหายถาวรหากระบบค้างกลางคัน
2. **Atomic Write Fallbacks (กันไฟล์ล็อคบน Windows):** หาก Windows Defender หรือ IDE (เช่น VS Code) ล็อคไฟล์ความจำไว้ ระบบ AI จะมีโหมดสลับไปเขียนทับโดยตรง (Fallback) ทันที ทำให้หมดปัญหา "บอทค้างตอนเซฟงาน"
3. **Cross-Platform Line Endings:** รองรับการจัดรูปแบบบรรทัดบน Windows (`\r\n`) สมบูรณ์แบบ ทำให้การอ่านและตัดข้อความแม่นยำ 100%

## 8. คำสั่ง CLI ทั้งหมดที่มีให้ใช้

นี่คือเครื่องมือที่คุณใช้ได้ผ่านคำสั่ง `./ai-protocol.sh [command]` ภายในโปรเจกต์ของคุณ:

| คำสั่ง | หน้าที่ | เมื่อไหร่ที่ควรใช้ |
|---|---|---|
| `init` | ติดตั้ง Protocol, คัดลอกเทมเพลต และสร้างไฟล์ IDE configs | ทำแค่ครั้งแรกตอนสร้างโปรเจกต์ใหม่ |
| `check` | ตรวจสอบสุขภาพโปรเจกต์ (เช่น ไฟล์ .env รั่วไหม, ไฟล์ RAM บวมไหม) | รันทุกวัน หรือก่อนจะ `git push` |
| `prune` | ตัดบันทึก Reflexions ที่เก่าเกินไปทิ้งไปเก็บในแฟ้มถาวร | เมื่อไฟล์ `REFLECTIONS.md` มีมากกว่า 15 รายการ |
| `clean` | แบ็คอัป `STATE.md` เดิมและสร้างอันใหม่ที่ว่างเปล่าให้ | เมื่อทำฟีเจอร์ใหญ่เสร็จ และจะเริ่มทำฟีเจอร์ใหม่ |
| `handoff` | สร้าง Prompt รวมบริบทเพื่อส่งไม้ต่อให้ AI แชทใหม่ | เมื่อแชท AI ยาวเกินไป หรือเริ่มคุยกันไม่รู้เรื่อง |
| `dashboard` | เปิดหน้าจอ AI Second Brain Hub เพื่อดูสรุปข้อมูลทั้งหมด | เมื่อต้องการดูภาพรวมว่า AI กำลังทำอะไร และมีความจำอะไรบ้าง |
| `install-hook` | ติดตั้ง Git Pre-commit Hook เพื่อบังคับรัน `check` อัตโนมัติ | ทำหลังจากสั่ง `git init` ครั้งแรก |
