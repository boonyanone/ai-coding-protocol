## 1. การติดตั้งและเริ่มต้นโปรเจกต์ (สำหรับมือใหม่)

การเริ่มต้นใช้งานนั้นง่ายมากครับ ขั้นตอนนี้คือการสร้าง "สมองความจำ" ให้กับ AI ในโปรเจกต์ของคุณ

**ขั้นตอน:**
1. เปิด Terminal (หรือหน้าต่าง Command ในโปรแกรมเขียนโค้ดของคุณ) แล้วพิมพ์คำสั่งเข้าไปที่โฟลเดอร์โปรเจกต์: `cd ~/my-app`
2. พิมพ์คำสั่งเริ่มต้นระบบ (Init) ลงไป:
   ```bash
   npx --yes github:boonyanone/ai-coding-protocol init
   ```
3. **เกิดอะไรขึ้นหลังจากนี้?** 
   - ระบบจะสร้างโฟลเดอร์ล่องหนชื่อ `.ai/` ขึ้นมา ซึ่งเปรียบเสมือนสมุดจดงานของ AI
   - มีการสร้างไฟล์ `STATE.md` เพื่อให้ AI จดว่ากำลังทำอะไรอยู่ จะได้ไม่ลืมงานเก่าๆ
   - ระบบจะบอกให้โปรแกรมเขียนโค้ดของคุณ (เช่น Cursor, Windsurf) ทำงานเป็นระเบียบมากขึ้น

**ทางเลือกเสริม:** ติดตั้ง Git Hook เพื่อป้องกันการอัปโหลดไฟล์ `.env`:
```bash
./ai-protocol.sh install-hook
```

## 2. การตั้งค่า Design System

กำหนดรูปแบบ UI ที่ชัดเจน เพื่อให้ AI เขียนโค้ดดีไซน์ออกมาตรงกันทุกครั้ง

**วิธีตั้งค่า:**
เลือกเทมเพลตจาก `.ai/templates/ui/` แล้วก๊อปปี้ไปไว้ที่ `.ai/docs/ui_guidelines.md`

### เทมเพลตระดับพรีเมียมที่มีให้เลือก:

#### 1. Shadcn UI (แนวมินิมอล สะอาดตา)
![Shadcn UI Mockup](https://raw.githubusercontent.com/boonyanone/agent-workflow/main/public/images/shadcn.png)
```bash
cp .ai/templates/ui/shadcn.md .ai/docs/ui_guidelines.md
```

#### 2. Aceternity UI (แนวล้ำยุค อวกาศมืด และ Glassmorphism)
![Aceternity UI Mockup](https://raw.githubusercontent.com/boonyanone/agent-workflow/main/public/images/aceternity.png)
```bash
cp .ai/templates/ui/aceternity.md .ai/docs/ui_guidelines.md
```

#### 3. NextUI (แนวสดใส พาสเทล)
![NextUI Mockup](https://raw.githubusercontent.com/boonyanone/agent-workflow/main/public/images/nextui.png)
```bash

```

#### 4. Blocks.so + shadcn/ui (สำหรับแดชบอร์ดระดับพรีเมียม)

หลังจากนี้ AI จะอ่านไฟล์ `ui_guidelines.md` เพื่อใช้เป็นแนวทางในการเขียน UI

## 3. ขั้นตอนการทำงานประจำวัน

เพื่อประหยัด Token และให้ AI ทำงานได้แม่นยำ ควรแบ่งงานเป็นส่วนเล็กๆ

**วิธีทำงาน:**
1. **เริ่มงาน:** เปิดแชทใหม่ คัดลอกข้อความใน `.ai/prompts/01-session-start.md` ไปแปะเป็นคำสั่งแรก เพื่อให้ AI ทราบสถานะโปรเจกต์
2. **สั่งงานย่อย:** สั่งให้ AI แก้ไขไฟล์ทีละน้อย (2-3 ไฟล์) ไม่ควรสั่งทำฟีเจอร์ใหญ่รวดเดียว
3. **ทดสอบ:** ทดสอบโค้ดบ่อยๆ หากเจอบั๊กให้สั่ง AI แก้ไขทันที
4. **จบงาน:** เมื่อฟีเจอร์ย่อยเสร็จ คัดลอกข้อความใน `.ai/prompts/02-session-end.md` ไปแปะ เพื่อให้ AI บันทึกความคืบหน้าลง `.ai/STATE.md`

## 4. การจัดการไฟล์ความจำ

ดูแลไฟล์ในโฟลเดอร์ `.ai/` ไม่ให้ใหญ่เกินไป

- **`STATE.md`:** ใช้บันทึกงานปัจจุบัน หากต้องการขึ้นงานใหม่ทั้งหมด ให้สำรองไฟล์เดิมและล้างข้อมูลใหม่ด้วยคำสั่ง:
  ```bash
  ./ai-protocol.sh clean
  ```
- **`REFLECTIONS.md`:** ใช้บันทึกบทเรียนที่เคยแก้บั๊ก หากมีเกิน 15 ข้อ ให้ย้ายข้อความเก่าไปเก็บถาวรด้วยคำสั่ง:
  ```bash
  ./ai-protocol.sh prune
  ```
- **`DECISIONS.md`:** ใช้บันทึกการตัดสินใจเรื่องสถาปัตยกรรม (เช่น เลือกใช้เครื่องมือ หรือ Database อะไร)

## 5. การย้ายแชท (Handoff)

เมื่อแชทเริ่มยาวและ AI เริ่มทำงานช้าลง ควรย้ายไปแชทใหม่โดยใช้เครื่องมือ Handoff เพื่อส่งต่อบริบทงาน

**ขั้นตอน:**
1. รันคำสั่ง Handoff:
   ```bash
   ./ai-protocol.sh handoff
   ```
2. คัดลอกข้อความที่ระบบสร้างให้
3. เปิดแชทใหม่ใน IDE
4. วางข้อความที่คัดลอกมาลงในแชทใหม่ เพื่อทำงานต่อจากเดิม

## 6. หน้าปัดสรุปงาน (Dashboard)

ดูสรุปสถานะของโปรเจกต์ผ่าน Terminal

```bash
./ai-protocol.sh dashboard
```
ระบบจะแสดงงานที่ค้างอยู่, บทเรียนล่าสุด, และการตัดสินใจเชิงสถาปัตยกรรม

## 7. การเชื่อมต่อ NotebookLM (ผู้ช่วยอ่านไฟล์ใหญ่)

เวลาเราให้ AI อ่านโค้ดหรือเอกสารยาวๆ มันจะกิน Token เยอะและทำให้ AI เอ๋อ (Context Bloat) เราจึงใช้ Google NotebookLM มาช่วยอ่านแทน

**ขั้นตอนการใช้งาน:**
1. **ติดตั้งระบบ:** พิมพ์คำสั่ง `./ai-protocol.sh install-mcp`
2. **ล็อกอินเข้าสู่ระบบ:** พิมพ์คำสั่ง `./ai-protocol.sh auth-mcp`
   - *💡 คำแนะนำสำหรับมือใหม่:* สคริปต์นี้มีความฉลาดมาก! หากคุณเปิด Google Chrome ทิ้งไว้และล็อกอินบัญชี Google อยู่แล้ว สคริปต์จะทำการ "ดูดคุกกี้ (Cookie)" เพื่อล็อกอินให้อัตโนมัติในเสี้ยววินาที โดยที่คุณไม่จำเป็นต้องกรอกรหัสผ่านอะไรเลย!
3. **เตรียมข้อมูลในเว็บ NotebookLM:** 
   - เข้าไปที่เว็บไซต์ NotebookLM (notebooklm.google.com)
   - สร้าง "สมุดโน้ต (Notebook)" ใหม่ขึ้นมา (เช่น ตั้งชื่อว่า *AI Coding Protocol*)
   - อัปโหลดไฟล์เอกสาร, โค้ด, หรือก๊อปปี้ข้อความยาวๆ ไปใส่ในสมุดโน้ตนั้น
4. **วิธีสั่งงาน AI ในแชท IDE:**
   - พิมพ์สั่ง AI โดยระบุชื่อสมุดโน้ตไปด้วย เช่น *"ช่วยเชื่อมต่อ NotebookLM แล้วเข้าไปดึงข้อมูลจากสมุดโน้ตชื่อ 'AI Coding Protocol' มาให้หน่อย จากนั้นเอาข้อมูลมาช่วยฉันอธิบายหน้าเว็บให้ที"*
   - AI จะทำการค้นหาและดึงความรู้แบบเจาะลึกจากสมุดโน้ตนั้นมาใช้ตอบคุณทันที!

## 8. สรุปคำสั่ง CLI

รันคำสั่งผ่าน `./ai-protocol.sh [คำสั่ง]`

| คำสั่ง | การใช้งาน |
|---|---|
| `init` | ติดตั้ง Protocol ลงในโปรเจกต์ |
| `check` | ตรวจสอบไฟล์ `.env` และขนาดไฟล์ความจำ |
| `prune` | ตัดบทเรียนเก่าๆ ไปเก็บถาวร |
| `clean` | สร้างไฟล์ `STATE.md` ใหม่สำหรับการเริ่มงานใหม่ |
| `handoff` | สรุปบริบทเพื่อส่งต่อไปยังแชทใหม่ |
| `dashboard` | แสดงหน้าสรุปสถานะโปรเจกต์ |
| `install-hook` | ติดตั้งตัวป้องกันการ commit ไฟล์ความลับ |
| `install-mcp` | ติดตั้งระบบเชื่อมต่อ NotebookLM |
| `auth-mcp` | ล็อกอินเข้าใช้งาน NotebookLM |
