## 🛠️ 1. การติดตั้ง & เริ่มต้นโปรเจกต์
**ใช้เมื่อ:** คุณสร้างโปรเจกต์ใหม่ หรือต้องการนำ AI Protocol ไปครอบโปรเจกต์เดิมที่มีอยู่แล้ว

**📝 ขั้นตอนการทำงาน:**
1. เข้าไปที่โฟลเดอร์โปรเจกต์ของคุณใน Terminal (เช่น `cd ~/my-app`)
2. สั่ง **`init`** เพื่อติดตั้งระบบ (พิมพ์ที่อยู่ของสคริปต์ให้ถูกต้อง):
   ```bash
   /path/to/ai-coding-protocol/ai-protocol.sh init
   ```
3. *(แนะนำ)* สั่ง **`install-hook`** เพื่อติดตั้งระบบตรวจสอบอัตโนมัติก่อน Git Commit:
   ```bash
   ./ai-protocol.sh install-hook
   ```

## 🎨 2. การควบคุมดีไซน์ UI (Design System)
**ใช้เมื่อ:** ป้องกันไม่ให้ AI สุ่มสี สุ่มฟอนต์ หรือออกแบบมั่วซั่ว (ต้องกำหนดทิศทางให้ AI อ่านเป็นแหล่งเดียว)

**📝 ขั้นตอนการทำงาน:**
เลือกเทมเพลตที่ชอบ แล้วคัดลอกไปทับไฟล์ `.ai/docs/ui_guidelines.md`
- 🌌 **Futuristic Space-Dark** (อวกาศ, โทนดำ): `cp .ai/templates/ui/futuristic.md .ai/docs/ui_guidelines.md`
- 🤍 **Minimalist & Clean** (เรียบหรู ขาว-ดำ): `cp .ai/templates/ui/minimal.md .ai/docs/ui_guidelines.md`
- 🌈 **Modern Vibrant** (สีสันสดใส): `cp .ai/templates/ui/vibrant.md .ai/docs/ui_guidelines.md`
- 📊 **Data-Heavy Dashboard** (เน้นกราฟ/ข้อมูล): `cp .ai/templates/ui/data_heavy.md .ai/docs/ui_guidelines.md`

## 🔄 3. วงจรการทำงานประจำวัน (Daily Workflow)
**เคล็ดลับ:** ต้องให้ AI ทำแบบ **"Micro-Stepping" (ทำทีละฟีเจอร์ย่อย)** ห้ามสั่งกวาดทีเดียวหลายๆ อย่าง เพื่อป้องกัน AI หลอน (Hallucination)

**📝 ขั้นตอนการทำงาน:**
1. **🎬 เริ่มงาน:** เปิดแชทใหม่ ก๊อปปี้ข้อความจาก `.ai/prompts/01-session-start.md` มาแปะ เพื่อตั้งค่าสมองเริ่มต้นให้ AI
2. **🧑‍💻 สั่งงาน:** สั่งทีละส่วน (แก้ 2-3 ไฟล์ต่อรอบ) เมื่อ AI บอกว่ารอตรวจ (`[WAITING_FOR_USER]`) ให้คุณรันทดสอบดู ถ้าพังให้สั่งแก้ทันที ห้ามสะสมบั๊ก!
3. **🏁 ปิดจ็อบ:** เมื่อฟีเจอร์ย่อยเสร็จ ก๊อปปี้ข้อความจาก `.ai/prompts/02-session-end.md` มาแปะให้ AI บันทึกความจำลงระบบ

> [!TIP]
> 🎯 **เมนูลัด IDE Task Menu (ไม่ต้องจำคำสั่งคลิกเอาได้เลย):** 
> หากใช้ VS Code, Cursor หรือโปรแกรมตระกูลเดียวกัน คุณสามารถไปที่เมนู `Terminal -> Run Task...` แล้วคลิกเลือกคำสั่งได้เลยโดยไม่ต้องพิมพ์ใน Terminal!

## 🧠 4. การจัดการสมองของ AI (Memory Management)
ระบบมีกล่องความจำ 3 ใบที่คุณต้องคอยดูแลไม่ให้รกเกินไป:

- **1️⃣ `STATE.md` (RAM / งานปัจจุบัน):** 
  - **ห้ามปล่อยให้ยาวเกิน 20 บรรทัด!** 
  - 🧹 **วิธีดูแล:** เมื่อทำฟีเจอร์ใหญ่เสร็จสมบูรณ์ ให้สั่ง **`clean`** เพื่อล้างกระดาน
- **2️⃣ `REFLECTIONS.md` (บทเรียนจากบั๊ก):** 
  - AI จะจดวิธีแก้ปัญหาไว้กันลืม **แต่ถ้ายาวเกิน 15 ข้อ Token จะบวม** 
  - ✂️ **วิธีดูแล:** สั่ง **`prune`** เพื่อตัดบทเรียนเก่าๆ ไปเก็บใน Archive
- **3️⃣ `DECISIONS.md` (การตัดสินใจโครงสร้าง):** 
  - ใช้จดสถาปัตยกรรม (เช่น ใช้ฐานข้อมูลอะไร) AI ห้ามเปลี่ยนเทคโนโลยีเองต้องถามคุณก่อนเสมอ

## 🏃 5. ย้ายแชทเมื่อ AI เริ่มเอ๋อ (Handoff)
**ใช้เมื่อ:** แชทเริ่มยาวเกินไป AI ทำงานช้าลง หรือเริ่มสับสน และคุณต้องการย้ายไปแชทใหม่เพื่อ **ทำฟีเจอร์เดิมที่ค้างอยู่ให้จบ**

**📝 ขั้นตอนการทำงาน:**
1. สั่ง **`check`** 👉 เพื่อดูความเรียบร้อย
2. สั่ง **`handoff`** 👉 เพื่อให้ระบบสรุปงานที่ค้างอยู่
3. **เปิดแชทใหม่** 👉 นำข้อความ Handoff ไปแปะ แล้วลุยงานต่อได้ทันที!
*(⚠️ ห้ามสั่ง `clean` เด็ดขาด เพราะงานยังไม่เสร็จ)*

## 📊 6. หน้าปัดควบคุม (Dashboard)
**ใช้เมื่อ:** ต้องการดูภาพรวมโปรเจกต์ภายใน 1 วินาที โดยไม่ต้องไปไล่เปิดอ่านไฟล์ `.ai/` ทีละไฟล์

**📝 ขั้นตอนการทำงาน:**
- สั่ง **`dashboard`**
- คุณจะเห็นสรุป 3 เรื่องทันที: 📍 งานที่ค้างอยู่, 📚 บทเรียนล่าสุด, 🏗️ การตัดสินใจสถาปัตยกรรม

## 🕵️ 7. คลังวิจัยเชิงลึก (Deep Research Hub & NotebookLM)
**ใช้เมื่อ:** คุณมีไฟล์ Document, API Reference, หรือคู่มือที่ **หนาเป็นร้อยเป็นพันบรรทัด** และไม่อยากโยนเข้า IDE ตรงๆ ให้เปลือง Token หรือทำให้แชท AI บวมและเริ่มเอ๋อ

**✨ ฟีเจอร์นี้คืออะไร? และช่วยคุณได้อย่างไร?**
เราได้เชื่อมต่อ **NotebookLM MCP** เข้ามาในระบบ เพื่อทำหน้าที่เป็น "คลังวิจัยสมองกล" ประจำโปรเจกต์!
- 💰 **ประหยัด Token ขั้นสุด:** แทนที่ AI ใน IDE จะต้องอ่านไฟล์ 10,000 บรรทัด มันจะใช้วิธี "แชทถาม" เอาเฉพาะข้อมูลที่จำเป็นจาก NotebookLM
- 🧠 **Context ไม่พัง:** แชทของคุณจะลื่นไหลเหมือนเดิม เพราะภาระการอ่านไฟล์ยักษ์ถูกผลักไปให้เซิร์ฟเวอร์ของ Google จัดการ
- 📚 **เชื่อมต่อแบบไร้รอยต่อ:** เมื่อติดตั้งแล้ว AI ใน IDE ของคุณสามารถสั่งสร้างสมุดโน้ต, อัปโหลดไฟล์/ลิงก์, และสกัดข้อมูลกลับมาเขียนโค้ดต่อได้ทันทีโดยไม่ต้องสลับหน้าจอไปมา!

**📝 วิธีติดตั้งและเรียกใช้งาน:**
1. **ติดตั้ง (ทำครั้งเดียว):** รันคำสั่ง `./ai-protocol.sh install-mcp` ใน Terminal ของโปรเจกต์
2. **ล็อกอิน:** รันคำสั่ง `node .ai/mcp/notebooklm/build/browser-auth.js` ตามที่ระบบแนะนำ เพื่อเชื่อมบัญชี Google ของคุณอย่างปลอดภัย
3. **วิธีเรียกใช้งาน:** ในช่องแชทของ Cursor/Windsurf ให้คุณสั่ง AI ไปตรงๆ เลยว่า:
   > *"ช่วยโหลด Document ของลิงก์ [ใส่ลิงก์ API] เข้าไปสร้างเป็นสมุดโน้ตใน NotebookLM ให้หน่อย จากนั้นสรุปวิธีใช้งาน Component นี้มาให้ที"*

## 🛡️ 8. ความปลอดภัยของข้อมูล (Fail-safe)
ระบบถูกออกแบบมาให้ **"ข้อมูลไม่มีทางพัง"**:
- **สำรองก่อนลบ:** คำสั่งอย่าง `prune` จะเซฟไฟล์ลง Archive 100% ก่อนถึงจะลบของเดิม
- **กันไฟล์ล็อค:** มีระบบ Atomic Write ช่วยแก้ปัญหาเวลา Windows หรือ IDE ล็อคไฟล์ ทำให้บอทไม่ค้างเวลาเซฟงาน
- **Cross-Platform:** รองรับบรรทัดทั้งแบบ Mac และ Windows อัตโนมัติ

## 💡 9. สรุปสูตรโกงคำสั่ง (CLI Cheat Sheet)
คำสั่งทั้งหมดใช้งานผ่าน `./ai-protocol.sh [command]`

| คำสั่ง | เมื่อไหร่ที่ต้องใช้? |
| :--- | :--- |
| **`init`** | 🚀 **ครั้งแรกครั้งเดียว** ตอนเริ่มเอา Protocol มาลงโปรเจกต์ |
| **`check`** | 🩺 ตรวจสุขภาพโปรเจกต์ (แนะนำให้รันก่อน Git Push หรือย้ายแชท) |
| **`handoff`** | ⏳ เมื่อมี **งานค้าง** แต่แชทเริ่มยาว/รวน เลยต้องย้ายแชทไปทำต่อ |
| **`clean`** | ✅ เมื่องาน **เสร็จสมบูรณ์** และต้องการล้างกระดานเพื่อทำฟีเจอร์ใหม่ |
| **`prune`** | ✂️ เมื่อไฟล์ความจำ (บทเรียน) เริ่มยาวเกิน 15 ข้อ ให้ตัดไปเก็บถาวร |
| **`dashboard`** | 📊 เมื่ออยากดูสรุปภาพรวมทั้งหมดว่า AI จำอะไรไว้บ้าง |
| **`install-hook`** | 🪝 ติดตั้งตัวเช็คก่อน commit (ทำครั้งเดียวหลัง `git init`) |
| **`install-mcp`** | 🕵️ **ครั้งแรก** เพื่อติดตั้งตัวเชื่อมต่อคลังวิจัย Deep Research (NotebookLM) |
